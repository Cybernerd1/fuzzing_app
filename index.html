What is Fuzzing?

Fuzzing is an automated process of identifying software vulnerabilities by supplying unexpected and faulty inputs to the software. It allows us to find the critical edge cases where software might fail, providing crucial insights into the stability and security of the system.

Step 1: Identification of Target Functions

Target functions are typically those functions that act as entry points for processing input data. They are vital to understand how the software handles different inputs.

Step 2: Developing Harness

A harness is a small code stub created to invoke the target function using mutated data inputs. It enables smooth integration between the fuzzer and the target software.

Step 3: Fuzzing

The fuzzer generates numerous data inputs that are passed to the target function through the harness. Any crashes or failures are documented for later analysis.

Challenge Description

In this challenge, teams must fuzz the Windows variant of the Sumatra PDF Reader (version 3.5.2 or later). The task involves:

  • Identifying target functions within the Sumatra PDF Reader
  • Developing a working fuzzing harness using appropriate inputs
  • Performing fuzzing using a tool like WinAFL
  • Providing a live demonstration and a technical report

Expected Solution

Each team is required to provide a fuzzing harness that targets the Sumatra PDF Reader software (v3.5.2 or later). The submission should include:

  1. A working fuzzing harness
  2. Steps undertaken to reverse-engineer the software
  3. List of identified target functions and dependencies
  4. A comprehensive technical report
