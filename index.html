Fuzzing Challenge: Sumatra PDF Reader

What is Fuzzing?

Fuzzing is an automated process of identifying software vulnerabilities by supplying unexpected and faulty inputs to the software.

Steps in Fuzzing Process:

  1. Identification of Target Function(s): Target functions are entry points for processing input data.
  2. Developing Harness: Harnesses are code stubs that invoke the target function with mutated inputs.
  3. Fuzzing: A fuzzer generates inputs, checks if the application crashes, and stores inputs and memory states for analysis.

Description of the Challenge:

The teams are required to develop a working harness for fuzzing the latest version (3.5.2 or later) of the Windows Sumatra PDF Reader. The harness should fuzz the software using a fuzzer such as WinAFL.

Evaluation Criteria:

  • Target functions identified
  • Live demonstration of the harness
  • Code coverage achieved
  • Technical report submission

Expected Solution:

The team must provide a fuzzing harness capable of fuzzing the Sumatra PDF Reader software version 3.5.2 or later. The harness must identify target functions and supply appropriate arguments. The fuzzer should be able to generate numerous inputs and store crash information for further analysis.

Technical Report Should Include:

  • Reversing steps undertaken
  • Target functions identified
  • Dependencies identified
